# Support

Use the right channel so maintainers can respond efficiently.

## Bug Reports

- Open a GitHub issue using the Bug Report template.
- Provide reproducible steps, expected behavior, and actual behavior.

## Feature Requests

- Open a GitHub issue using the Feature Request template.
- Explain the problem, the proposed solution, and tradeoffs.

## Usage Questions

- Open a GitHub issue with title prefix `[Question]` and include:
  - what you are trying to achieve
  - environment details
  - what you already tried

## Security Issues

For security concerns, do not use public issues. Follow [SECURITY.md](SECURITY.md).
